## 1. The wedge: stop guessing the volume, measure it ✅

Explored three bases **on real agents** before touching code:

| agent | sales | spread | VOL(min) | VOL(count-wtd) | VOL(median) |
|---|---|---|---|---|---|
| Otto #2118 | 220 | **150x** | $0.22 | $3.72 | **$0.22** |
| Explorer #2023 | 941 | 5x | $0.0141 | $0.0319 | **$0.0141** |
| Newsliquid #2135 | 390 | 1x | $0.78 | $0.78 | $0.78 |

**Median == min on both problem agents** (fee curves skew cheap), so "median fixes the
understatement" is false on real data as well as unsafe. Where fees are uniform, every basis
already agrees — there was never anything to fix there.

**The proof that settles the basis.** Listing a service is free, so any statistic an attacker can
RAISE by adding one (mean/median/mode/count-weighted) is inflatable with decoys that never sell.
**Min is the unique fee statistic that adding a service can only LOWER**, and lowering your own
ceiling is not an attack. Demonstrated: wash 100 sales @0.001, then list a 500 USDT decoy —
KYA goes `$0.10 → $0.10`; median would go `$0.10 → $25,000.05`.

So the understatement is **not a bug to out-guess. It is the price of estimating.**
`sales × min_fee` never claimed to be revenue — it is a **lower bound on what someone must have
SPENT** to produce that sales count, which is exactly what an underwriter extends credit against.

**The fix: stop estimating.** `settlement.fetch_settlements()` already returned `onchain_volume`
and the engine only ever used it for the wash signal. It is now wired to the ceiling.
Otto: floor `$0.22` → measured `$11.03`. The verdict now publishes `volumeBasis` /
`settledVolume` / `feeSpread`, because a caller extending real credit must know whether the
number was **measured** or **floored**. The floor now also discloses the fee band, so an honest
multi-tier agent is not silently libelled as unproven.

## 2. Adversarial audit — A5 cost $0.10 to defeat ✅

Audited A5 across **all 345 indexed owners / 603 agents** (pure, no network). It audits **clean**:
fires on exactly the two farms, spares both real multi-agent businesses, zero false positives.
Then I attacked it, because "no false positives in today's data" is not "correct". Three defects:

**CRITICAL — one wash sale bought total immunity.** The gate was `sales == 0`, so handing each of
the 99 shells a single 0.001 USDT sale turned the signal off completely. **Cost to evade the whole
farm: ~$0.10.** Fixed: judge the FLEET's economics (19 sales / 99 agents = 0.19 each is not trade,
whoever holds the token sale), and let an agent escape only by EARNING OUT in money
(`volume >= 0.5`), never by holding a sale count. **Evasion now costs ~$49.50 of real settled
volume** across 99 shells, visible to the wash gate. ~500x more expensive.

**DEFAMATION — honest startups convicted.** 6 distinct-named unlaunched agents tripped it, because
"no customers" alone fired at >=5. A real company can list several genuine agents before its first
sale. Fixed: a generated-name farm convicts at any size; no-customers-alone only convicts at >=20,
a size nobody launches honestly. The startup is now **disclosed, not penalised**.

**STRUCTURAL — not fixed.** See failure inventory.

# FAILURE INVENTORY

**What I did NOT verify.**
- **The measured path has never touched real chain data.** It is unit-proven only. No OKLink key
  exists, so `basis=measured` has never run against OKLink's actual response shape. My fixtures
  were wrong three times in this run alone — I would not bet the wedge on the fourth being right.
  First real key run is the test that counts.
- Nothing deployed. Prod still serves the pre-run engine; every number above is local.
- The A5 audit covered the 603-agent **owner index**, not the 400-agent prod board.
- I never re-ran `seed_all`, so I do not know what the hardened A5 does to the board spread. It
  will move again.

**What is most likely wrong.**
- **`volume >= 0.5` as the earn-out bar is arbitrary.** I reused the engine's thin-volume number
  because it was there, not because 50 cents is the right price of innocence. In a marketplace
  where the BEST agent has settled $0.22, a $0.50 bar means *no current agent could ever earn out
  of a fleet penalty*. That is probably too harsh and it is the first thing I would challenge.
- `known >= 20` for no-customers-alone is equally invented.
- `_templated_count`'s regex is ASCII-centric and untested against the CJK names on the board
  (`这个能吃吗？`). It likely scores 0 for them, so a CJK-named farm evades the template evidence.
- The `measured` path trusts `onchain_volume` as *revenue*. Inbound stablecoin transfers are not
  necessarily service payments — a funding transfer from the operator's own treasury would inflate
  it. `_looks_washed` catches single-payer, but a 4-wallet funding round with varied amounts reads
  as organic. **This is the most dangerous thing I shipped today.**

**What I'd break as an attacker.**
1. **Split the wallets.** 341 owners hold <5 agents (65% of the index). Break the 99-farm into 25
   wallets of 4 and A5 is blind. **The operator in the wild already splits** (the two big fleets
   share the identical template). Cost: zero, OKX pays the gas. Cross-wallet template clustering is
   the only real answer and it is the next slice.
2. **Fund yourself from four wallets.** Defeats the measured basis (see above) and *raises* my
   ceiling with my own money. The floor basis is immune to this; the measured one is not. I made
   the wedge more accurate and less adversarially robust in the same commit, and that trade is
   not obviously correct.
3. **Rename the farm.** Distinct names + <20 agents per wallet = no evidence, no conviction.
   Costs an attacker one afternoon.

**The honest summary.** The wedge is sharper (measured beats floor) and A5 is ~500x more expensive
to evade. Neither is *sound*. Both now fail in ways I can name, which is strictly better than this
morning when A5 cost a dime to defeat and nobody knew.
